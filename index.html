C++中指向成员函数的指针

2013-11-28

在C++中，类的非静态成员函数因为含有隐含的this指针，不同于普通函数。所以C++引入了一个新的指针类型：指向成员函数的指针。
注意：类的static成员函数和普通函数的指针是一样的。

  • 普通函数指针：可以通过直接调用 
int f(int x); //the type is "int (*)(int)"
  • 非静态成员函数指针：只能通过类对象调用 
int Mat::f2(int x); //the type is "int (Mat::*)(int)"

指向成员函数的指针和普通指针，是两种不同类型指针，C++实现机制是不一样的（例如成员函数指针不一定包含具体函数的内存地址），如果强制进行两者之间的转换，结果是未定义的

Item28 Smart pointer

  • 智能指针相比于内置指针的优势：
    1.构造和析构：分别防止空指针（默认实参是0）和内存泄露（dtor内delete对象）
    2.复制和赋值：可以定义出3种语义：深复制、浅复制(加引用计数、或转移控制权)、不允许复制
    3.解引用(dereference)：包括->和*操作符(都是const函数)，可以实现出lazy fetching（数据库中的对象的访问）

2013-11-06

Item 26: Postpone variable definitions as long as possible.

1.延后变量定义的意义是，有可能变量定义后，出现异常导致变量用不到，这样变量的构造和析构函数就白费了。所以，只在使用这个变量的代码之前定义它。
2.循环内变量的使用，是个特殊情况。但是仍然鼓励在循环内，变量使用前才定义变量。除非满足下面两点
(1)赋值成本比构造+析构成本低 (2)该段代码是效率敏感（performance-sensitive）的部分
因为：缩小变量的作用域，对于程序的可理解性和易维护性好。

Item 30: Understand the ins and outs of inlining.

1.inline（函数体代替函数调用）的好处：
（1）没有函数调用的开销，速度快。
（2）使编译器能对起进行outlined的函数所不具备的优化，因为上下文更明显了。
坏处：
（1）可能导致代码膨胀，如果inline函数内语句比较多的话。
（2）如果inline函数是程序库的一部分，对它的改动，需要使用者重新编译。（outline的话，直接链接即可，动态链接的话，使用者甚至不知道函数库的修改）
（3）大部分调试器不能调试inline函数。
2.inline是对编译器的申请，不是强制。

（1）函数太过复杂（包含递归或循环）。
