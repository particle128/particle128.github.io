题目：给你int rand()，等概率返回0或1，要求实现int randn(int n)，等概率返回0~n-1

那天的情景，中间穿插失败原因分析：

  1. 第一印象有点蒙，一直觉得自己数学不咋好，这个貌似跟数学有关系，一开始就怯了
    -> 太依赖leetcode了，以为面试都是leetcode的变形呢，措手不及。其实网上面试题中有类似的。
  2. 想了1min，就开始提出很幼稚和没经过思考的想法，“调用n-1次rand，然后相加，行不行？”
  3. 他说不行，举了个例子给我，“rand3中0、1、2的概率不相等，1出现的概率大”
  4. 又想了1min，还没想清楚，就说“用一个数组，保存1...n-1的数，然后每次shuffle一个元素，然后...”
  5. 他说“想清楚再说，不要有点想法，就问我这个行不行，那个行不行”
    -> 此处暴露自己不独立思考，思路不清晰。
  6. 想了几min，说“如果n是2是幂次方，我倒是会做”，然后给他讲了一下思路，证明的确是等概率的。
  7. 他赞同我的观点，然后问我“如果n不是2的幂次方呢”
  8. 想了几min，记得网上说遇到不会的可以跳过，就问“我数学不太好，能不能换一个题呢？还是必须先做完这道题？”
    -> 此处暴露了自己畏难，这是面试的大忌
  9. 他不太满意，说“这个也没用到什么数学知识，顶多高中的数学。。。假设这就是工作中，我遇到了这么一个问题，跟你探讨，你要怎么帮我解决”
  10. 无奈，只能硬着头皮想，好久还是没想出来。
  11. 25min过去了，面试官让一筹莫展的我写一下“2的幂次方”的代码。
  12. 10行就写完了，他说“看起来没啥问题，但是如果参数是负数或0呢，这个代码可以么？”
  13. 我就说，本来这个代码就是限制的很死，需要参数给2的幂次方，就没考虑负数或0的情况，就像有些库函数一样，参数不对结果未定义。因为如果要考虑输入的合法性，需要很多防御编程的代码
    -> 后来我才知道，google很看重代码的健壮性，根本不像那本leetcode题解上说的“不鼓励防御式编程”。即使很难检查参数合法性的时候，也要提前告诉面试官这种情况，让他知道你注意到参数合法性问题了。
  14. 他就说“算了，那你利用这个函数2次幂的函数，考虑一下能不能实现任意n吧”
    -> 这其实是提示我，这个函数是前提，可以基于它弄，可惜当时脑子不转，还是不会。
  15. 还是不会...最后，他告诉我一种思路，问我可不可以，我说可以，并解释了一下为什么可以。我提出找一个很大的数字，直接MAX%n，就像C语言库函数那样，当时他说可以，然后面试就结束了。
    -> 一般有两道题或者一道题要求优化，我第一道就没做出来，肯定没戏了。最后提出的想法，其实也不对，只是当时时间仓促，他没注意，事后肯定也发现了。

我事后写的答案:

int rand2() {
    return rand()%2;
}
int _randn(unsigned int n) {  // assume n = 2^x
    int bits = -1;
    while (n) {
        bits++;
        n >>= 1;
    }
    int res = 0;
    for (int i = 0; i < bits; ++i) {
        res  = (res<<1) | rand2();
    }
    return res;
}
int randn(int n) {
    if (n<=0)   return -1;
    unsigned int _n = 1; // the smallest power of 2 that is bigger than n
    while (_n < n) {
        _n <<= 1;
    }
    int res;
    while (true) {
        res = _randn(_n);
        if (res < n)
            return res;
    }
}

改进：

  • 准备要更充分。不能光刷leetcode，还要刷网上的各种面经，还有cc150。准备越充分，现场越从容。
  • 面试技巧非常重要。不要畏难、沟通、解题从简单到最优、边界。 这里讲的技巧很好

1、在线编程测试

简历不刷人，直接在线测试。2.5小时，4道题。结果：前两道拿了100，后两道一点没做。

题目类型：

  1. 不需要算法，纯代码题，注意一些特殊case
    -> WA了3次才AC，共用时1h20min。很像google测试的第一题的风格，代码量50~100行，需要考虑的case很多，没有算法
  2. 考察栈和数组的数据结构灵活运用
    -> 一次AC，用时40min
  3. 需要两轮DP
    -> 当时只想到第一轮DP的写法，第二轮DP不会写，提前20min放弃比赛
  4. 需要线段树
    -> 题目都没看

总结：

  • 在线测试前，没过多看题，挺放松的，可以让自己减少紧张程度，并且思路不局限
  • 平时写代码，注意提高自己的思维速度，并且考虑全面各种case再写
  • DP需要专门训练，才能运用更灵活，比如01背包、多轮DP、数位DP等

AA面试的流程是这样的：早上8点开始，首先介绍流程，然后是45min的笔试，3轮45min的技术面试，最后是45min的小组讨论。我只有第二轮技术面试是中文的，其他全程英语。值得一提的是，面试需要穿正装。他们正式IT员工，工作时也是西装、衬衣，金融机构果然不一样。

  2. 一面：自我介绍、讲解笔试题目、追加题目要求、讲解实习的项目
  3. 二面：自我介绍、做题（给定四个点，判断是正方形）
  4. 三面：自我介绍、问了sql、问了智力题（要在北京到上海建火车，问应该造几辆火车）
  5. 小组讨论：三人一个小组、后来6人汇合成一个大组、6个人做5min的presentation
