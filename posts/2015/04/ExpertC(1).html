第四章 令人震惊的事实：数组和指针并不相同

数组/指针在不同源文件中的定义和声明，应该形式一样。

一个源文件下定义的数组int a[10]，在另一个源文件下声明，必须使用extern int a[]形式；
一个源文件下定义的指针int *a，在另一个源文件下声明，必须使用extern int *a形式。

原因：同样一条语句a[0] = 1;，编译器对于a是数组和a是指针的处理方式不同。

假设我们定义了

int a[10] = {0};

并且a在符号表中的内容为：

var_name | var_address
a        | 123456

在外部用两种不同方式声明：

extern int a[]; a[1] = 1;

a是外部变量的声明，不分配空间，编译时用Undefined符号代替。
链接时，查找符号表，找到a，把符号表中a的地址123456当做数组首地址，把1赋给内存单元(123456+4)

extern int *a; a[1] = 1;

a是外部变量的声明，不分配空间，编译时用Undefined符号代替。
连接时，查找符号表，找到a，把符号表中a的地址123456当做指针地址，取(123456)内存单元中的内容，假设是654321，把该数值当做数组的首地址，然后把1赋给内存单元(654321+4)。
