Index compression

2016-06-26

This article is a note of Chapter 4 in《这就是搜索引擎》.

Basic ideas

The meanings of index compression are:

  • Reduce SSD and memory occupation of index, which can save machines.
  • Provide opportunities for increasing index size, which usually can bring relevance gain.

The basic principle is "Trade time for space". All algorithms here will need more memory operations than no index compression, but compared with the large space reduction, that little time increase can be ignored.

There are several ways to achieve this.

Dictionary compression

A very primitive structure of the leaf node in a dictionary (B-tree or hash table). It contains document frequency (DF), word, and a pointer to a posting list. To make "word" segment be able to accommodate all words, it has to be of 40B. For short words, such as "cat", 37B are wasted.

A better choice. In this structure, every "word" segment contains a pointer to the real word. To decrease the memory fragmentation, we allocate all words in a sequential space one by one, with '\0' as the delimiter. The following diagram shows 2 nodes.

To go further, we combine 2 (or other number) nodes, to decrease 1 pointer(8 bytes for 64-bit OS). For every word in the "word block", the word size must precede the first letter of the word, so that we can know where the next word is.

Posting list compression

In general, posting list is a list of doc. The basic node of the posting list contains 2 parts, doc index and term frequency (TF). Both of them are integer numbers. So, the goal of posting list compression is to compress a list of integer numbers.

There are many algorithms to achieve this, such as Variable byte, SimpleX, and PForDelta. Among those, PForDelta has the fastest decompression speed, which makes it most popular.

PForDelta

The basic idea is

  1. Every adjacent N numbers are grouped together. Usually, N = 1024.
  2. Select top largest M% numbers. These numbers will not be compressed. Usually, M = 10.
  3. For the other 100-M numbers, choose the smallest bit length (K) that can accommodate the largest number. Every number in this scope will bit use K bits to store.
  4. Store the index information in the original position for non-compressed large numbers.

Example

Suppose N = 10, M = 3, the numbers are 24,40,9,13,31,67,19,44,22,10.
